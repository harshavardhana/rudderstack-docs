# Destinations

A **destination** is a cloud tool or platform where you want to send the data via RudderStack.

<div class="successBlock">

RudderStack currently supports over 100 <a href="https://rudderstack.com/docs/destinations/">destinations</a>. These include <a href="https://rudderstack.com/docs/data-warehouse-integrations/">data warehouses</a>, <a href="https://rudderstack.com/docs/destinations/crm/">CRMs</a>, <a href="https://rudderstack.com/docs/destinations/analytics/">analytics</a> and <a href="https://rudderstack.com/docs/destinations/marketing/">marketing</a> platforms, and more.
</div>

RudderStack Cloud provides **Destinations** option in the left navigation bar that allows you to view all the configured destinations in your RudderStack dashboard:

<img src="../assets/general/rudderstack-destinations.png" alt="RudderStack destinations" />


## Adding a destination

Follow these steps to add a destination in RudderStack:

1. Log in to the **RudderStack dashboard**.

<img src="../assets/1 (26).png" alt="RudderStack Dashboard" />
<span class="imageTitle"></span>

2. Click **Destinations** in the left navigation bar and then click on **New Destination**. 


<img src="../assets/1 (27).png" alt="Add Destination" />
<span class="imageTitle"></span>

<div class="infoBlock">
Alternatively, you can add a destination directly connected to your source from the source details window as shown:

<img
  src="../assets/latest-1.png"
  alt="Connect your source to a destination"
/>
<span class="imageTitle"></span>
</div>

3. Click on the destination you want to set up in RudderStack. 
4. Assign a name to the destination and click on **Next**.
5. Click on **Add Source** to add a new source to be connected with this destination. Refer to the detailed steps for [Adding a source]().

Alternatively, you can select an existing source (if any) for this destination and click on **Next**.

<img
  src="../assets/screen-shot-2021-05-19-at-4.54.30-pm.png"
  alt="Select the data source"
/>
<span class="imageTitle"></span>

6. Configure the destination with the relevant connection settings. Refer to the specific [**destination documentation**](https://rudderstack.com/docs/destinations/) for more details on these settings.

<img src="../assets/ga1 (1).png" alt="Configure the destination" />
<span class="imageTitle"></span>

7. Click on the **CREATE NEW TRANSFORMATION** option if you want to add a [**transformation**](https://rudderstack.com/docs/transformations/). Otherwise, click on **Next**.

<img
  src="../assets/screen-shot-2021-05-19-at-5.01.56-pm (1).png"
  alt="Create new transformation"
/>
<span class="imageTitle"></span>

Your destination should be configured successfully.

<img src="../assets/latest23.png" alt="Configuration successful" />

### Destination details

To get details about a configured destination, click on it to see following view on the dashboard:

<img src="../assets/general/rudderstack-destination-details.png" alt="RudderStack destination details" />

These options can be explained as follows:

- **Rename**: Click on the edit icon next to the destination name to rename the destination:

<img
  src="../assets/screen-shot-2021-08-25-at-11.18.08-am.png"
  alt="Rename a destination"
/>
<span class="imageTitle"></span>

- **Sources**: It displays all the sources connected to a destination and their status (enabled/disabled). You can also connect a new source by clicking on the **Add source** button.

- **Transformation**: You can add an existing transformation to the destination via this option.

<div class="infoBlock">

To add a new transformation, go to the <strong>Transformations</strong> option in the left navigation bar and click on <strong>New Transformation</strong>.
</div>

- **Events**: You can get all the latest metrics related to the event delivery such as number of events successfully delivered, failed events, etc. You can also filter these by source and time period.

- **Configuration**: This option lets you change the existing configuration of a destination.

- **Settings**: This option gives you additional destination-related details, such as:
  
  - **Enabled**: You can enable or disable sending data to the destination via this option.
  - **Destination ID**: This is the unique identifier associated with your destination.
  - **Permanently delete destination**: You can delete the destination by clicking on the **Delete** button.

<div class="warningBlock">

Before you delete a destination, make sure there are no sources connected to it.
</div>

### Cloning a destination

You can duplicate an existing destination with the same settings by following these steps:

1. Click on the **Duplication Destination** option in the **Destination Details** page:

<img src="../assets/general/duplicate-destination-1.png" alt="Duplicate a destination" />

2. Assign a name to the new destination and click on **Next**.

<img src="../assets/general/duplicate-destination-2.png" alt="Assigning a name to the duplicate destination" />

3. Connect the destination to a source:

<img src="../assets/general/duplicate-destination-3.png" alt="Connecting a new source to duplicate destination" />

The new destination is successfully configured.
